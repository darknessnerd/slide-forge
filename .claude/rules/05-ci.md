# CI

## Gate Sequence

Run gates in this order. Each must pass before the next runs.

| Order | Gate | Command | Blocks merge |
|-------|------|---------|-------------|
| 1 | Format | `gofmt -l . \| grep . && exit 1` or `gofumpt -l .` | Yes |
| 2 | Vet | `go vet ./...` | Yes |
| 3 | Lint | `golangci-lint run ./...` | Yes |
| 4 | Unit tests | `go test ./...` | Yes |
| 5 | Race detector | `go test -race ./...` | Yes |
| 6 | Coverage | `go test -coverprofile=coverage.out ./... && go tool cover -func=coverage.out` | Yes — see thresholds below |
| 7 | Integration tests | `go test -tags=integration ./test/integration/...` | Yes (requires running deps via testcontainers) |

## Coverage Thresholds

No authoritative Go standard prescribes a numeric floor — thresholds here are team decisions.
These match the targets in `03-testing.md`:

- **Total:** 80% minimum
- **Per package (`domain/`, `service/`):** 100% of exported functions tested (enforced by test review, not tooling)
- **Per file:** no automated floor — coverage report reviewed in PR

Enforce in CI with [`go-test-coverage`](https://github.com/vladopajic/go-test-coverage):

```bash
go test -coverprofile=coverage.out ./...
go-test-coverage --config=.testcoverage.yml
```

## Linters

Configuration lives in `.golangci.yml`. Enabled set:

**Non-negotiable (team baseline):**
- `errcheck` — unchecked errors are bugs; check-type-assertions and check-blank enabled
- `govet` — reports suspicious code patterns (printf format, struct alignment, etc.)
- `staticcheck` — comprehensive Go static analysis (replaces megacheck)
- `unused` — reports unused exported identifiers
- `gosec` — security checks (hardcoded creds, weak crypto, SQL injection, etc.)
- `errorlint` — prevents incorrect `errors.Is`/`errors.As` usage
- `ineffassign` — detects assignments whose values are never used
- `gofumpt` — stricter gofmt, enforces consistent formatting (or `goimports`)

**Recommended additions:**
- `revive` — opinionated Go style rules (replaces golint)
- `gocritic` — 100+ code smell checks; enable selectively
- `bodyclose` — flags unclosed HTTP response bodies
- `noctx` — flags `http.NewRequest` without context
- `exhaustive` — enforces exhaustive enum switches

**Do NOT enable without team approval:**
- `wrapcheck` — requires all errors to be wrapped; generates excessive false positives on entry points
- `ireturn` — enforces returning interfaces; conflicts with our convention of returning concrete types from constructors
- `varnamelen` — low signal-to-noise in short-lived function scopes

## Merge Requirements

A PR may not merge unless all of the following pass:

1. All 7 CI gates green (no skips, no manual overrides)
2. `go mod tidy` has been run — `go.mod` and `go.sum` must be clean
3. No new `// nolint` directives on security-related linters without team approval (see `04-security.md`)
4. Race detector clean — zero races reported
5. At least one reviewer approval

## Running Locally (same commands as CI)

```bash
# Full pre-push check — mirrors CI gates 1-6
gofumpt -l . && go vet ./... && golangci-lint run ./... && go test -race -coverprofile=coverage.out ./...

# Integration tests (requires Docker for testcontainers)
go test -tags=integration -race ./test/integration/...

# Lint only
golangci-lint run ./...

# Fix auto-fixable lint issues
golangci-lint run --fix ./...
```
